Things

Things is a personal notepad for whatever is currently occupying your head: a thought, a reminder, a plan, a mood, a place, a person, a future version of yourself trying not to forget something small but important.

Each entry gets a name, a date, optional tags, and the ability to be starred when it matters more than the rest. The interaction model is clean and opinionated: swipe left to mark something complete, then find finished items later on their own searchable page.

What I like here is the restraint. This is not trying to become a sprawling productivity system. It is closer to a curated shelf for half-formed commitments — light enough to use quickly, structured enough to stay useful after the moment passes.

Under the hood it is a native SwiftUI iOS app with no third-party dependencies, JSON persistence through UserDefaults, and a project setup that leans into reproducibility. Even the app icon and Xcode project file are generated by scripts, which is a very specific kind of developer neatness I respect.

View code
Things/ ├── Models/Thing.swift # model + date helpers + grouping ├── Models/Persistence.swift # UserDefaults JSON store ├── Theme/Theme.swift # violet-accent design system ├── Views/HomeView.swift # active list, swipe to complete ├── Views/CompletedView.swift # finished items, undo / delete └── Views/EditorView.swift # create and edit flow

It feels like the kind of product that comes from caring about the shape of everyday cognition: how to capture, tag, revisit, and quietly close loops without turning the process into admin. Less task manager. More mental desk.
